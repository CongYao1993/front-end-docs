1. JS 异常监控

JS 异常监控是为了提高感知、定位、处理的效率。

  • 开发者迅速感知到 JS 异常发生
  • 通过监控平台迅速定位问题
  • 能够高效的处理问题，追踪问题的处理进度，并统计问题

2. 错误采集方式

2.1 try/catch

只能捕获代码常规的运行错误，不能捕获语法错误和异步错误

// 常规运行时错误，可以捕获 ✅
 try {
   let a = undefined;
   if (a.length) {
     console.log('111');
   }
 } catch (e) {
   console.log('捕获到异常：', e);
}

// 语法错误，不能捕获 ❌
try {
  const notdefined,
} catch(e) {
  console.log('捕获不到异常：', 'Uncaught SyntaxError');
}

// 异步错误，不能捕获 ❌
try {
  setTimeout(() => {
    console.log(notdefined);
  }, 0)
} catch(e) {
  console.log('捕获不到异常：', 'Uncaught ReferenceError');
}
